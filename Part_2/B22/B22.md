## B22 – Enhance the cybersecurity of a website from your community

### Objective

The objective of this activity was to improve the cybersecurity of a real community website by implementing a simple but effective security enhancement. The goal was to identify a weakness in the website and apply a practical solution to reduce security risks.

---

### Findings

For this activity, I worked on improving the security of the website [Wholesome Tales](https://www.wholesometales.com/). While reviewing the website, I noticed that the “Contact Us” page contained a form submission feature but did not have any CAPTCHA protection enabled. This meant that the form was vulnerable to spam submissions, automated bots, and potentially malicious requests. Since contact forms are publicly accessible, they are often targeted by spambots if no verification system is present.

To fix this issue, I implemented Google reCAPTCHA on the contact form through the WordPress admin portal. The first step involved reviewing the current state of the contact page and confirming that no CAPTCHA was present.
**Evidence 1:** Screenshot showing the original contact page without CAPTCHA.

After that, I accessed the spam protection settings in the WordPress admin dashboard to review the available CAPTCHA options that could be integrated into the website.
**Evidence 2:** Screenshot showing the spam security settings section in WordPress.

I then used the Google reCAPTCHA portal to register the website domain and configure reCAPTCHA for the form.
**Evidence 3:** Screenshot showing the Google reCAPTCHA setup portal where the website was added.

Once the website was registered, Google generated a site key and secret key, which are required for integration with the website.
**Evidence 4:** Screenshot showing the generated site key and secret key.

These keys were then added into the WordPress admin portal so the website could communicate with Google’s reCAPTCHA service correctly.
**Evidence 5:** Screenshot showing the site key and secret key added into WordPress.

After configuration, I edited the contact form settings and enabled reCAPTCHA protection on the form itself.
**Evidence 6:** Screenshot showing that reCAPTCHA was enabled in the form editor.

Finally, the updated contact page displayed the Google reCAPTCHA verification feature successfully, confirming that the implementation was working correctly.
**Evidence 7:** Screenshot showing the new contact page with reCAPTCHA enabled.

Overall, this enhancement significantly improved the security of the contact form by reducing the risk of spam bots and automated abuse.

---

### Mitigations

To further improve the website’s security, the website administrator should continue monitoring form submissions for suspicious behaviour and keep all WordPress plugins updated regularly. Additional protections such as rate limiting, email filtering, and security plugins could also help reduce abuse attempts. It is also important to use strong administrator passwords and enable multi-factor authentication for the WordPress admin account to improve overall website security.

---

### Reflection

This activity showed me that even simple security improvements can make a meaningful difference to a real website. Before this task, the contact form had no protection against automated spam, which is a very common issue on smaller community websites. Implementing Google reCAPTCHA was not technically very difficult, but it immediately improved the security of the website in a practical way. It also helped me better understand how website security is often about reducing common risks through proper configuration and maintenance rather than only focusing on advanced attacks.
