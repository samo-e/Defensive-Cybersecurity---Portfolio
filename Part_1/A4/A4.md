## A4 – Discover a vulnerable website

### Objective

Identify a real-world website that shows at least one security weakness using only legal and non-intrusive methods. The goal is to understand how common website security issues appear in real life and what kind of risk they can create.

---

### Findings

* The website assessed was **happyfacesnewark.com**, which appears to be a public daycare/learning centre website with basic pages and a contact page for visitors. It looks like a normal small business site with no obvious advanced security features visible from the front end.

* While reviewing the site, it was clear that it can be accessed using **HTTP instead of HTTPS**, which immediately indicates weak transport security. This means the communication between the user and the website is not properly encrypted.

* The site also includes a **contact page/form**, which increases the risk because users may enter personal information such as names, emails, or phone numbers. If this is done over an insecure connection, that data could potentially be exposed.

* Overall, the security setup of the website seems basic, and important protections that are commonly expected on modern websites are either missing or not clearly enforced.

---

### Vulnerabilities

* The website is accessible over **HTTP instead of HTTPS**, meaning data is not encrypted during transmission.

* There is **no clear enforcement of HTTPS**, so users are not automatically redirected to a secure version of the site.

* A **contact form is available over an insecure connection**, which could expose user-submitted data.

* Likely absence of **HSTS (HTTP Strict Transport Security)**, which means browsers are not forced to use secure connections.

---

### Mitigations (high-level recommendations)

The main improvement would be to properly enable HTTPS across the entire website and ensure all HTTP traffic is automatically redirected to HTTPS. A valid TLS certificate should be installed and configured correctly. After that, HSTS should be implemented so that browsers always connect securely in future visits. Since the site includes a contact form, all pages that collect user data must only operate over HTTPS. It would also help to review basic security configurations to make sure the site follows standard modern security practices.

---

### Reflection

This example shows that even small websites can miss very basic security measures like HTTPS. Even though the site itself looks simple, the risk is still important because real users may be entering personal information. What I found interesting is that the vulnerability is not complex, but it can still have a real impact. It also shows that small improvements in configuration can make a big difference in overall security.
