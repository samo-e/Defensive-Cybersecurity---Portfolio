## A7 – Discover cryptography used in modern networks

### Objective

The objective of this activity was to identify different ways cryptography is used in modern networks and understand how it protects data when it is being sent across networks like the internet or private systems. I focused on common network technologies such as HTTPS, VPNs, Wi-Fi, key exchange, and certificates because these are the most visible examples of network cryptography in daily use.

---

### Findings

* **TLS in HTTPS connections**

  One of the most common uses of cryptography in networks is TLS (Transport Layer Security), which is used in HTTPS connections. It encrypts data between a user's browser and a server so that information like passwords or personal details cannot be easily intercepted while being transmitted. This is something we use almost every day without really noticing it.

  I noticed that TLS is critical for secure web browsing, especially on sites handling sensitive information. Without it, data would be exposed to anyone monitoring the network traffic.

* **IPsec in VPNs**

  IPsec is another cryptographic protocol used in networks, especially in Virtual Private Networks (VPNs). It secures communication between devices by encrypting and authenticating data packets. This is commonly used in corporate networks where secure communication between offices is needed.

  In my view, IPsec is important for remote work because it creates a secure tunnel over untrusted networks. It protects both confidentiality and integrity of the data being sent.

* **Wi-Fi encryption (WPA2/WPA3)**

  Wi-Fi networks use encryption protocols like WPA2 or WPA3 to protect wireless communication. These protocols ensure that only authorised users with the correct password can connect to the network and that the data being sent over Wi-Fi is encrypted.

  This is especially relevant for public Wi-Fi hotspots, where anyone nearby could potentially intercept traffic. WPA3 is stronger than WPA2 because it provides better protection against brute-force attacks.

* **Public key cryptography for key exchange**

  Public key cryptography is used in network communication to securely exchange keys between two parties. For example, when establishing a secure connection, one party can share a public key which is used to encrypt data, and only the matching private key can decrypt it. This helps in setting up secure sessions.

  I found this interesting because it solves the problem of how two parties can agree on a secret key without sending it directly over the network. It is used in protocols like TLS during the handshake.

* **Digital certificates for identity verification**

  Digital certificates are used in networks to verify the identity of servers and services. These certificates are issued by trusted authorities and are used during secure connections to confirm that the user is communicating with the correct and legitimate server.

  This prevents man-in-the-middle attacks, where an attacker pretends to be the real server. Certificates make sure users can trust who they are connecting to.

---

### Mitigations

To keep cryptographic systems in networks secure, it is important to use modern protocols like TLS 1.2 or TLS 1.3 and avoid outdated ones. Strong encryption settings should be configured properly, and certificates should be valid and regularly updated. Network administrators should also ensure that secure Wi-Fi protocols like WPA3 are used, and weak passwords are avoided.

Overall, keeping systems updated and correctly configured makes a big difference. Sometimes people use old protocols because they are easier to set up, but this creates unnecessary risks.

---

### Reflection

This task made me realise how much cryptography is involved in everyday network communication. Most of the time, users don't even see it, but it plays a really important role in keeping data safe while it is being transferred. It also shows that using outdated protocols or poor configuration can create serious risks.

I also understood that network cryptography works best when multiple layers are used together. For example, TLS plus certificates plus good Wi-Fi encryption provides much better protection than any single method alone.
