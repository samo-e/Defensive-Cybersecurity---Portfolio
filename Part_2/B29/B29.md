## B29 – Find a CVE in this year and fix it using three different generative AI systems, comparing the consistency

### Objective

The objective of this activity was to select a recent CVE from this year, analyse the vulnerability, and compare how three different generative AI systems respond when asked to suggest fixes or mitigations. The goal was to evaluate the consistency, accuracy, and usefulness of AI-generated cybersecurity guidance.

---

### Selected CVE

For this activity, I selected **CVE-2026-20841**, a remote code execution (RCE) vulnerability related to Microsoft Windows Notepad’s Markdown handling feature. The vulnerability allowed attackers to potentially execute malicious code by tricking users into interacting with specially crafted Markdown links. The issue was caused by improper validation of links when rendering Markdown files. ([Zero Day Initiative][1])

This CVE was chosen because:

* it is recent and well documented,
* it affects a widely used application,
* and the vulnerability has clear mitigation steps that can be compared across AI systems.

---

### AI Systems Used

The three generative AI systems used for comparison were:

1. ChatGPT
2. Google Gemini
3. Microsoft Copilot

Each AI system was given a similar prompt asking:

* what the vulnerability does,
* how it can be mitigated,
* and what actions users or administrators should take.

---

### Findings

All three AI systems correctly identified the vulnerability as a remote code execution issue related to malicious Markdown links in Notepad. Each system also consistently recommended updating Windows and applying Microsoft’s official security patches as the primary fix. This showed a strong level of consistency across the AI tools regarding the most important mitigation step.

ChatGPT provided the most structured explanation overall. It clearly explained the attack scenario, the role of Markdown links, and why patching the system was important. It also suggested avoiding suspicious Markdown files and restricting untrusted content until updates were applied. The response was detailed but still understandable for non-expert users.

Google Gemini produced a similar response and also emphasised updating Windows immediately. Its explanation was slightly shorter and more concise, focusing more on the official mitigation rather than the technical details of the exploit. However, the recommendations were still accurate and aligned with public advisories.

Microsoft Copilot gave responses that were strongly aligned with Microsoft security guidance and patch management recommendations. Since the vulnerability affected Microsoft software, Copilot focused heavily on installing the latest Windows updates and reducing user interaction with untrusted Markdown content. The explanation was less technical but practical from a user perspective.

Although all three systems gave similar core recommendations, there were some differences in depth and clarity. ChatGPT generally provided more contextual explanation, Gemini focused on concise summarisation, and Copilot leaned more towards official patch guidance and operational advice.

Overall, the responses were largely consistent regarding:

* the nature of the vulnerability,
* the severity of the issue,
* and the need for immediate patching.

This consistency increases confidence that modern AI systems can provide reasonably accurate high-level cybersecurity guidance for known vulnerabilities.

---

### Reflection

This activity showed me that generative AI systems can provide useful cybersecurity guidance, especially for explaining vulnerabilities and recommending basic mitigations. However, the quality and detail of responses still vary between systems. Some models explain concepts more clearly, while others mainly summarise official guidance.

It also made me realise that AI should not completely replace official advisories or expert judgement. Even though the AI systems were mostly consistent, users still need to verify information using trusted security sources such as vendor advisories, NVD entries, or government cybersecurity guidance. AI is useful for assistance and learning, but human verification is still very important in cybersecurity.
