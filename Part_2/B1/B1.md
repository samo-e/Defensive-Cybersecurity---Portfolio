## B1 – Discover 5 unique weak/vulnerable security implementations

### Objective

The objective of this activity was to identify five different examples of weak or vulnerable security implementations that are still commonly seen in real life. The goal was to understand how poor security practices can create risks for users, systems, and organisations.

---

### Findings

* One common weak security implementation is using **HTTP instead of HTTPS** on websites. Without HTTPS, data transmitted between the user and the website is not encrypted, which means attackers on the same network may be able to intercept sensitive information such as passwords or personal details. Even today, some smaller websites still do not properly enforce HTTPS.

* Another weak implementation is the use of **default passwords** on routers, IoT devices, or security cameras. Many devices are shipped with simple usernames and passwords like “admin/admin,” and users often forget to change them. This makes it very easy for attackers to gain unauthorised access.

* **Weak password policies** are also a major issue in many systems. Some websites still allow extremely simple passwords or do not require multi-factor authentication. This increases the risk of brute-force attacks and account compromise, especially when users reuse passwords across different platforms.

* Outdated software or systems that do not receive regular updates are another example of vulnerable implementation. Older software versions may contain known vulnerabilities that attackers can easily exploit. This is especially dangerous in businesses where systems are not patched regularly.

* Poor physical security can also be considered a weak security implementation. For example, leaving server rooms unlocked or allowing unrestricted access to office areas increases the risk of theft, tampering, or unauthorised access to sensitive systems and devices.

---

### Mitigations

These weak implementations can be improved by following basic cybersecurity best practices. Websites should enforce HTTPS, default passwords should always be changed immediately, and systems should require strong passwords with multi-factor authentication. Software should be updated regularly to patch known vulnerabilities, and physical access to important systems should be restricted using proper locks or access control systems. Overall, even simple security improvements can greatly reduce risk.

---

### Reflection

This activity showed me that many security problems happen because of simple mistakes or weak configurations rather than highly advanced attacks. In many cases, organisations already have security tools available, but they are not configured properly or maintained well. It also made me realise that security needs to be considered in both digital and physical environments together.
