## A8 – Discover cryptography used in Internet of Things (IoT) devices

### Objective

The objective of this activity was to identify different ways cryptography is used in IoT devices and understand how small, connected devices (like smart home devices or wearables) protect data and communication. I focused on common IoT examples such as smart thermostats, security cameras, fitness trackers, and smart locks because these are popular in homes and offices.

---

### Findings

* **Lightweight encryption like AES**

  Many IoT devices use lightweight encryption (like AES) to protect data because they have limited processing power. Since these devices are small and not very powerful, they cannot handle heavy encryption, so lighter versions are used to still keep data secure while saving resources.

  I noticed that AES is common in IoT because it is fast and secure enough for most sensor data. For example, a smart door lock might use AES to encrypt its status before sending it to your phone.

* **Secure communication protocols (TLS/DTLS)**

  Secure communication protocols like TLS or DTLS are used in IoT devices to encrypt data sent between the device and servers. For example, a smart thermostat sending temperature data to a cloud server will use encryption to make sure the data cannot be easily intercepted.

  This is important because IoT devices often send data over Wi-Fi, which is easy to eavesdrop on. DTLS is especially designed for small devices that need real-time communication.

* **Device authentication with keys/certificates**

  Device authentication is an important use of cryptography in IoT. Each device is given a unique key or certificate so that it can prove its identity before connecting to a network or server. This helps prevent unauthorised devices from joining the system.

  In my view, this stops attackers from spoofing legitimate devices. For instance, a fitness tracker uses its unique key to authenticate with the manufacturer's server before syncing your health data.

* **Firmware signing with digital signatures**

  Firmware signing is used to ensure that updates installed on IoT devices are legitimate. Before installing an update, the device checks a digital signature to confirm that the firmware has not been tampered with, which helps prevent malicious software from being installed.

  This is critical because IoT devices are often not updated often. A security camera, for example, verifies firmware signatures to avoid getting infected with malware.

* **Hashing for data integrity**

  Hashing is used in IoT devices to verify data integrity. When data is sent or stored, a hash value is created and later checked to make sure the data has not been changed or corrupted.

  I found this useful for things like sensor readings. A smart bulb might hash its configuration data to detect if someone tried to modify it.

---

### Mitigations

To keep IoT devices secure, it is important to use strong but efficient encryption methods that suit low-power devices. Default passwords should always be changed, and firmware should be updated regularly to fix vulnerabilities. Proper key management is also important, as weak or exposed keys can compromise the entire system.

Additionally, using secure communication protocols and isolating IoT devices on separate networks can reduce risk. Sometimes people connect everything to the main network, which is not safe.

---

### Reflection

This task helped me understand that IoT devices also rely heavily on cryptography, even though they are small and simple. Since these devices are often connected all the time, weak security can create big risks. It shows that even basic devices need proper protection to keep data and networks safe.

I also realised that IoT cryptography needs to balance security with performance. Heavy encryption would drain batteries or slow down devices, so lightweight methods are designed specifically for this environment.
