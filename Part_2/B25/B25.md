## B25 – Design and implement a threat intelligence module of your choice

### Objective

The objective of this activity was to design and implement a basic threat intelligence and protection system for a real website. The goal was to improve the website’s ability to detect, monitor, and block malicious activity before it could affect the system.

---

### Findings

For this activity, I implemented a website security and threat monitoring solution on the website [Wholesome Tales](https://www.wholesometales.com/) using the Wordfence Security plugin developed by Mark Maunder. Before this task, the website did not have any firewall or active website protection system enabled, which meant there was limited protection against attacks such as malicious traffic, login attempts, bots, or exploit requests.

I first reviewed the Wordfence plugin through its official WordPress plugin page to understand its firewall and monitoring capabilities. The plugin provides features such as a web application firewall (WAF), attack monitoring, login security, and live traffic analysis.
**Evidence 1:** Screenshot showing the Wordfence Security plugin information page on WordPress.org.

After installing and configuring the plugin, I reviewed the website traffic information through the WordPress admin portal. This allowed me to see how much activity and traffic the website was receiving and helped demonstrate why traffic monitoring is important for identifying unusual behaviour or suspicious requests.
**Evidence 2:** Screenshot showing website traffic statistics in the WordPress admin portal.

Once Wordfence was fully enabled, the plugin started monitoring and filtering traffic automatically. The firewall began blocking suspicious requests and potential attacks targeting the website. The Wordfence dashboard displayed summaries of blocked attacks and malicious traffic attempts, which confirmed that the protection system was actively working.
**Evidence 3:** Screenshot showing the Wordfence dashboard with blocked attack statistics and security summaries.

Overall, this implementation improved the security posture of the website significantly because it introduced active monitoring, filtering, and attack prevention mechanisms that were previously missing.

---

### Mitigations

To further improve the website’s security, the Wordfence plugin should be updated regularly so that it can detect the latest threats and vulnerabilities. Website administrators should also monitor security alerts and blocked traffic logs to identify repeated attack attempts. Additional protections such as strong passwords, multi-factor authentication, regular backups, and keeping WordPress plugins updated should also be maintained. Combining firewall protection with good website management practices provides stronger overall security.

---

### Reflection

This activity helped me understand the importance of proactive security and real-time monitoring in website protection. Before this task, the website had no firewall, which meant malicious requests could potentially reach the site without filtering. Implementing Wordfence showed me how threat intelligence tools can automatically detect and block suspicious activity before it becomes a bigger issue. It also gave me practical experience with using security plugins and understanding how modern websites monitor attacks in real time.
